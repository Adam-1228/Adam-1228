## 🧭 Principles

1. **데모가 아니라 운영** — 발표용 결과물이 아니라 매일 돌아가는 시스템을 만든다.
2. **관찰 가능성 우선** — 로그 · 지표 · 알림을 먼저 설계한다. 보이지 않는 시스템은 고칠 수도 없다.
3. **HITL로 신뢰** — 중요한 결정에는 사람이 개입할 여지를 남긴다. 속도보다 신뢰가 먼저.
4. **비용을 먼저 설계** — API · 인프라 · 실패 비용을 설계 단계에서 계산한다. 그래야 오래 돌아간다.

진단 (1일)  →  설계 (1주)  →  구현 (2–4주)  →  운영 (상시)
